**Product Requirements Document**
Версия: 1.0 | Дата: 2026-04-29 | Статус: Draft

---

## 1. Обзор продукта

### Проблема
Во время драфта в Dota 2 сложно быстро оценить оптимальный пик — нужно учесть контрпики против всех врагов, синергию с союзниками, текущую мету и роли. Вручную это занимает время, которого в драфте нет.

### Решение
Telegram-бот, который принимает список вражеских героев (и опционально союзных), анализирует данные с нескольких Dota 2 ресурсов и выдаёт топ-5 рекомендаций с детальным объяснением.

### Целевой пользователь
Игроки Dota 2 любого уровня, которые хотят принимать более обоснованные решения в драфте.

## 3. Источники данных

| Сайт | Что берём | Приоритет |
|------|-----------|-----------|
| **Dotabuff** | Matchups (контрпики), Duos (синергии), общий winrate | Основной |
| **Dota 2 Pro Tracker** | Пики профи игроков, мета тренды, популярность героя | Дополнительный |
| **Stratz.com** | Углублённая статистика, winrate по патчам, роли | Дополнительный |
| **OpenDota API** | Бесплатный API, данные по матчам, статистика героев | Резервный / API |

### Что парсим с каждого источника

**Dotabuff** (`dotabuff.com/heroes/{hero}/matchups`):
- Advantage % против каждого врага
- Win Rate в матчапе

**Dotabuff** (`dotabuff.com/heroes/{hero}/duos`):
- Win Rate в паре с союзниками

**Dota 2 Pro Tracker**:
- Частота пика в про-играх
- Winrate в текущей мете у профи

**Stratz / OpenDota API**:
- Общий winrate героя в текущем патче
- Популярность по ролям (позиция 1-5)

## 4. Алгоритм подбора пика

```
Входные данные:
├── Враги: [Hero1, Hero2, Hero3, Hero4, Hero5]
└── Союзники (опционально): [Ally1, Ally2, ...]

Для каждого кандидата-героя считаем:

Score = (Counter Score × 0.40)
      + (Synergy Score × 0.25)
      + (Meta Score × 0.20)
      + (Pro Pick Score × 0.15)

Фильтрация:
├── Исключить дублирующиеся роли (если уже есть керри — не предлагать второго)
└── Исключить героев с winrate < 48% в патче

Вывод: топ-5 героев по итоговому Score
```

### Расшифровка весов

| Компонент | Вес (без профиля) | Вес (с профилем) | Источник |
|-----------|------------------|-----------------|---------|
| Counter Score | 40% | 30% | Dotabuff matchups — среднее advantage против всех 5 врагов |
| Synergy Score | 25% | 20% | Dotabuff duos — winrate в паре с союзниками |
| Meta Score | 20% | 15% | Общий winrate в патче (Stratz / OpenDota) |
| Pro Pick Score | 15% | 10% | Частота и winrate в про-играх (Dota 2 Pro Tracker) |
| Player Score | — | 25% | Личный winrate и количество игр на герое (Dotabuff профиль) |

## 5. Профиль игрока (Dotabuff)

Пользователь может привязать свой Dotabuff профиль через Steam ID. После привязки бот учитывает личную статистику при подборе пика.

### Что берём из профиля

| Данные | Откуда | Зачем |
|--------|--------|-------|
| Winrate на герое | `dotabuff.com/players/{id}/heroes` | Рекомендовать только тех, кого игрок умеет играть |
| Количество игр на герое | Dotabuff профиль | Фильтр: герои с < 10 игр помечаются как "новые" |
| Последние матчи | `dotabuff.com/players/{id}/matches` | Исключить героев, которых играли последние 2 игры (антитильт) |
| Любимые роли | Dotabuff профиль | Приоритизировать пики под предпочтительную позицию |

### Логика Player Score

```
Player Score для героя =
  (личный winrate на герое × 0.6)
+ (количество игр / 100, макс 1.0) × 0.4

Если игр < 10 — герой помечается ⚠️ "мало опыта"
Если игр = 0 — герой помечается 🆕 "не играл"
```

### Команды для профиля

| Команда | Описание |
|---------|---------|
| `/profile {Steam ID или Dotabuff URL}` | Привязать профиль |
| `/profile` | Показать текущий привязанный профиль |
| `/profile remove` | Отвязать профиль |

### Пример вывода с профилем

```

⚔️  Контрпики: Invoker (+14%), CM (+11%)
🤝 Синергия: Shadow Fiend (+6%)
📊 Мета: WR 54.2% | Про-пики: 68%
👤 Ваша статистика: 58% WR | 127 игр
🎭 Роль: Midlane (поз. 2)

━━━━━━━━━━━━━━━━━━━━
2. Anti-Mage  |  Score: 79/100  ⚠️ мало опыта (3 игры)
```

## 6. Архитектура бота

```
Пользователь (Telegram)
        │
        ▼
Telegram Bot (python-telegram-bot)
        │
        ├── Parser Module
        │   ├── dotabuff_parser.py     ← matchups + duos + профиль игрока
        │   ├── protracker_parser.py   ← про-мета
        │   ├── stratz_parser.py       ← патч-статистика
        │   └── opendota_api.py        ← API (резерв)
        │
        ├── Player Profile Module
        │   └── profile.py             ← хранение Steam ID, кэш статистики игрока
        │
        ├── Cache Module
        │   └── cache.json             ← TTL 6 часов, не парсить каждый запрос
        │
        ├── Scoring Engine
        │   └── scorer.py              ← алгоритм подбора пика (с/без профиля)
        │
        └── Response Formatter
            └── formatter.py           ← красивый вывод в Telegram
```

## 10. Кэширование данных

- Данные по каждому герою кэшируются на **6 часов**
- При первом запросе — парсинг всех источников
- При повторном запросе — из кэша (мгновенно)
- Принудительное обновление: `/refresh`

---

## 11. Ограничения v1.0

- Поддерживаются только английские названия героев
- Парсинг зависит от структуры сайтов (может сломаться при редизайне)
- Нет авторизации — бот публичный
- Нет учёта skill bracket пользователя (рекомендации для всех рангов)
